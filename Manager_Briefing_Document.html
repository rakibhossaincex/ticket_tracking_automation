Ticket Tracking Automation System

Manager Briefing Document

Executive Summary

This document provides a complete overview of our automated ticket tracking and analytics system. The system automatically collects resolved ticket data from Intercom, stores it in a centralized database, and presents actionable insights through an interactive dashboard.

Key Benefits:

  • Automated Data Collection: No manual data entry required
  • Real-time Analytics: Dashboard updates every 2 hours
  • Performance Visibility: Track team and agent performance at a glance
  • SLA Monitoring: Identify missed SLAs and resolution bottlenecks

System Architecture

┌─────────────┐ ┌─────────────┐ ┌─────────────┐ ┌─────────────┐ │ Intercom │ ──► │ n8n │ ──► │ Supabase │ ──► │ Dashboard │ │ (Source) │ │ (Workflow) │ │ (Database) │ │ (Analytics)│ └─────────────┘ └─────────────┘ └─────────────┘ └─────────────┘

The system consists of three main components:

  1. n8n Workflow - Fetches data from Intercom API
  2. Supabase Database - Stores and manages ticket data
  3. Analytics Dashboard - Visualizes insights

Component 1: n8n Workflow (Data Collection)

What is n8n?

n8n is an automation platform that connects different services. Think of it as a "robot" that runs tasks automatically on a schedule.

How the Workflow Works

  1. Scheduled Trigger: Every 2 hours, the workflow automatically starts
  2. Intercom API Call: Fetches all resolved tickets from the last 24 hours from these teams:
    • Pro Solutions Task Force
    • CEx Reversal
    • Ticket Dependencies
    • Payments and Treasury
    • GB Email Communication
    • Tech Team
    • Business Operations
    • Platform Operations
    • Case Resolution
  3. Data Processing: For each ticket, extracts:
    • Ticket ID and creation date
    • Handler (agent who resolved it)
    • Team assignment
    • Resolution time
    • Issue category
    • Last ticket note (description)
  4. SLA Calculation: Determines if the ticket met or missed its SLA based on:
    • Category-specific time limits
    • Working hours (9 AM - 6 PM)
    • Weekends excluded
  5. Database Insert: Sends processed data to Supabase

Key Files

  • All Team Ticket log.json - The workflow configuration file

Component 2: Supabase Database (Data Storage)

What is Supabase?

Supabase is a cloud database service. It stores all our ticket data securely and makes it accessible to the dashboard.

Data Stored

Column Description
ticket_id Unique Intercom conversation ID
date When the ticket was resolved
ticket_handler_agent_name Agent who handled the ticket
current_team Team the ticket belongs to
resolution_time How long it took to resolve (e.g., "2h 15m")
sla "Met", "Missed", or "N/A"
issue_category Type of issue (e.g., "Breach Event", "Server Issue")
product_type CFD or Futures
description_last_ticket_note Summary of the ticket

Why Supabase?

  • Free tier for up to 500MB of data
  • Real-time updates - Dashboard always shows latest data
  • Secure - Data is encrypted and access controlled

Component 3: Analytics Dashboard (Visualization)

Overview

The dashboard is a web-based interface that displays ticket data in an easy-to-understand format. It's hosted on GitHub Pages and accessible via browser.

URL: https://rakibhossaincex.github.io/ticket_tracking_automation/

Dashboard Sections Explained

Header Section

  • Refresh Data: Manually reload data from Supabase
  • Last Updated: Shows when data was last fetched
  • Sync Note: "Data fetched every 2 hours from Intercom"

Filters

  • Date Range: Filter by specific dates
  • Handler: Select one or multiple agents (multi-select)
  • Team: Select one or multiple teams (multi-select)
  • SLA Status: Filter by Met, Missed, or N/A
  • Search: Free text search across all fields

How Data Flows (End-to-End)

1. Customer raises ticket in Intercom ↓ 2. Agent resolves the ticket ↓ 3. n8n workflow runs every 2 hours ↓ 4. Workflow fetches resolved tickets from Intercom API ↓ 5. Calculates SLA status for each ticket ↓ 6. Stores data in Supabase database ↓ 7. Dashboard reads from Supabase ↓ 8. Manager views real-time analytics

Key Metrics Definitions

SLA (Service Level Agreement)

  • Met: Ticket resolved within the allowed time
  • Missed: Ticket took longer than allowed
  • N/A: Cannot calculate (e.g., no category or missing data)

SLA Time Limits (by Category)

Category Type Time Limit
Standard issues 24 hours
Urgent issues 4 hours
Complex issues 48 hours

Working hours only (9 AM - 6 PM, Mon-Fri)
